Aquitecturas Web

Las arquitecturas web son estructuras que definen cómo se organizan y se comunican los diferentes componentes de una aplicación web.

star_half Tipos de arquitectura

  • Arquitectura C/S Clásica - Múltiples Páginas

    Es la típica estructura Cliente/Servidor, Cada vez que el usuario interactúa (ej. hace clic), se recarga toda la página desde el servidor, Se usa en sitios tradicionales como portales de noticias o universidades.

  • Arquitectura C/S Multicapa - Múltiples Páginas

    Similar a la anterior, pero organizada en capas (presentación, lógica de negocio y datos), Mejora el mantenimiento y escalabilidad del sistema, Cada acción aún recarga una nueva página desde el servidor.

  • Arquitectura C/S Multicapa - Página Única SPA

    Toda la aplicación carga en una sola página web, Usa JavaScript para actualizar contenido sin recargar la página, Muy rápida y fluida para el usuario (como Gmail o Facebook).

  • Arquitectura C/S Multicapa - SPA / MPA- Hibridas

    Combina lo mejor de SPA y MPA, Algunas partes son páginas únicas (SPA) y otras se recargan por completo (MPA), Ideal para apps grandes donde no todo necesita ser SPA.

  • Arquitectura Hexagonal

    También llamada “puertos y adaptadores”, Divide la app en un núcleo central (lógica) y adapta todo lo externo (UI, base de datos, APIs), Favorece la independencia de tecnologías y pruebas más fáciles.

Servidores web, Servicio Hosting y Cloud Hosting

A continuación se define cada uno de los conceptos:

  • Servidor web: Es un software o hardware que entrega páginas web al navegador del usuario cuando este las solicita.
  • Servicio de hosting: Es el alquiler de espacio en un servidor para almacenar y publicar sitios web.
  • Cloud hosting: Es un tipo de hosting que usa múltiples servidores en la nube, ofreciendo mayor escalabilidad y disponibilidad.

terminal Softwares Utilizados por servidores web

  • Apache HTTP Server: Servidor web de código abierto, muy usado por su flexibilidad y compatibilidad con múltiples lenguajes como PHP, Perl y Python.
  • Nginx: Servidor web y proxy inverso ligero, ideal para manejar alto tráfico con excelente rendimiento.
  • Microsoft IIS: Servidor web desarrollado por Microsoft, integrado con Windows Server, compatible con aplicaciones .NET y ASP.
  • LiteSpeed: Servidor web comercial conocido por su velocidad, bajo consumo de recursos y compatibilidad con Apache.
  • Apache Tomcat: Servidor de aplicaciones web especializado en ejecutar aplicaciones Java con soporte para Servlets y JSP.

star_half Tipos de Hosting

  • Hosting compartido: Varios sitios web comparten los recursos de un mismo servidor. Es económico, pero puede ser lento si otros sitios consumen muchos recursos.
  • Hosting de servidor privado virtual (VPS): Un servidor físico dividido en varios entornos virtuales independientes. Ofrece más control y recursos que el hosting compartido.
  • Hosting dedicado: Todo el servidor físico está reservado para un solo usuario o empresa. Ofrece máximo rendimiento y control, ideal para sitios con mucho tráfico.
  • Hosting administrado: El proveedor se encarga de la configuración, mantenimiento y seguridad del servidor. Recomendado para quienes no tienen conocimientos técnicos avanzados.
  • Hosting en la nube: Usa una red de servidores conectados para garantizar disponibilidad y escalabilidad. Es flexible, seguro y se adapta a la demanda del sitio.
