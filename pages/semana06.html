¿Qué aprendí?

Eventos, condicionales y bucles en React

Caracteristicas:

  • Eventos: Manejan acciones del usuario (ej. onClick).
  • Condicionales: Muestran u ocultan elementos según condiciones (? :, &&).
  • Bucles: Usan .map() para renderizar listas dinámicas.

Eventos en React

Los eventos en React se manejan usando camelCase y funciones manejadoras.


                    import React from 'react';

                    function EventoEjemplo() {
                        const mostrarAlerta = () => {
                        alert('¡Hiciste clic!');
                    };

                    return (
                        <div>
                            <h2>Ejemplo de Evento</h2>
                            <button onClick={mostrarAlerta}>Haz clic aquí</button>
                        </div>
                    );
                    }

                    export default EventoEjemplo;

Condicionales en React

React no tiene estructuras if dentro del JSX, así que se usan expresiones como:

Operador ternario

                    {isLoggedIn ? <p>Bienvenida</p> : <p>Inicia sesión</p>}
Evaluación lógica

                    {isAdmin && <button>Panel de admin</button>}

Bucles en React

Para iterar sobre arrays, se usa .map() dentro del JSX.


                    const productos = ['Polo', 'Zapatillas', 'Gorra'];

                    function App() {
                    return (
                        <ul>
                            {productos.map((item, index) => (
                                <li key={index}>{item}</li>
                            ))}
                        </ul>
                    );
                    }

Fragmentos

Cuando devuelves varios elementos sin un div, usa <>...:


                    <>
                        <h1>Título</h1>
                        <p>Texto</p>
                    </>

Consumo de APIs

Es el proceso de obtener datos de un servidor (por ejemplo, desde una API REST) y mostrarlos en tu componente React. Se usa comúnmente fetch() o librerías como axios.

Tipos de APIs

  • Según su acceso
    • API Pública: Cualquiera puede usarla sin restricciones (a veces con una clave).
    • API Privada: Solo está disponible para uso interno de una empresa.
    • API De socios: Compartida con terceros autorizados por contrato.
  • Según su propósito
    • API Web: Permite que diferentes sistemas se comuniquen por Internet.
    • API de Sistema Operativo: Accede a funciones del sistema como cámara, archivos, GPS.
    • API De librerías o frameworks: Expone funciones listas para usar.
    • API de Base de datos: Permite conectarse y hacer consultas SQL desde otros sistemas.
  • Según su tipo tecnico
    • REST (Representational State Transfer): Usa HTTP (GET, POST, PUT, DELETE). Formato típico, JSON.
    • SOAP (Simple Object Access Protocol): Basado en XML. Más seguro y estructurado, pero más pesado.
    • GraphQL: Permite consultar solo los datos que necesitas. Más eficiente que REST en muchos casos.
    • gRPC (Google Remote Procedure Call): Basado en Protobuf. Rápido, ideal para microservicios.

Funcionamiento:

  • useEffect() se ejecuta cuando el componente se monta.
  • fetch() obtiene los datos desde la API.
  • Se guardan los datos en el useState().
  • Se renderiza la lista en JSX.

Ejemplo practico usando fetch() y useEffect


                    import React, { useState, useEffect } from 'react';

                    function App() {
                        const [usuarios, setUsuarios] = useState([]);

                        useEffect(() => {
                            fetch('https://ejemploapi.com/api/usuarios')
                            .then(respuesta => respuesta.json())
                            .then(data => setUsuarios(data))
                            .catch(error => console.error('Error al cargar los datos:', error));
                        }, []);

                        return (
                            <div>
                            <h2>Usuarios</h2>
                            <ul>
                                {usuarios.map(usuario => (
                                <li key={usuario.id}>{usuario.name}</li>
                                ))}
                            </ul>
                            </div>
                        );
                    }

                    export default App;
